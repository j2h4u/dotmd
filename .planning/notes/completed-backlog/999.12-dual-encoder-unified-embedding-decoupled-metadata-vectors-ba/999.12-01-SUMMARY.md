## Decisions Made

1. **VecComponentStore connection ownership**: `conn` is passed in by caller (pipeline.py); VecComponentStore does not open/close it. `store()` and `delete_by_entity_ids()` do not commit — caller is transaction owner. Consistent with EmbeddingCache pattern.

2. **Canonical path invariant as caller responsibility**: VecComponentStore does not normalize entity_ids. The module docstring explicitly documents that for the `meta` component, callers must pass `str(Path(fp).resolve())`. This keeps the class simple and puts the invariant where it can be enforced by the caller that has the path context.

3. **meta_checksum excludes kind**: Intentional by design. Kind changes are detected by `chunk_checksum` (kind is in chunk_checksum), which triggers full re-chunking that recomputes e_meta anyway. Documented in ADR in the function docstring.

4. **embedding_weights validator requires both "text" and "meta"**: Codex MEDIUM concern addressed — accepting arbitrary component names would silently omit a required encoder from fusion. Hard failure at startup is preferable to silent wrong results at query time.

5. **embed_checksum preserved**: Not removed here; removal is Plan 02 task (pipeline refactor). Removing it now would break the existing pipeline.py import before Plan 02 lands.
