<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Copied SQLite snapshot -> migration manifest | Snapshot data is trusted only as read-only migration evidence and must not trigger recomputation. |
| Graph export / feedback export -> Surreal target | Exported rows can contain arbitrary identifiers and metadata that must remain data-only and must be bounded by source_capture_manifest evidence. |
| Operator target URL / overwrite policy -> target writes | The runner can mutate a Surreal target only after explicit mode, safety gate, and overwrite checks. |
| Migration phase checkpoints -> success report | Checkpoints decide whether a migration is safe to treat as applied or restore-required. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-41-06 | Tampering / Data Loss | `run_surreal_migration()` target writes | mitigate | Default overwrite policy refuses non-empty targets; explicit replacement records target pre-counts and policy before any deletion. |
| T-41-07 | Tampering | Source refs and graph identifiers | mitigate | Continue using `SurrealRecordIdCodec` for record IDs and keep raw identifiers in explicit payload fields only. |
| T-41-08 | Information Disclosure | Migration reports carrying source refs, feedback messages, and graph properties | mitigate | Reports must be written only to caller-provided paths in Plan 41-03 and include redaction/sample controls; this plan structures report data so callers can redact if needed. |
| T-41-09 | Denial of Service | Long imports over production-derived data | mitigate | Use named phases and checkpoint counts so long-running imports can fail with phase evidence; no per-request or service-start import path is added. |

| T-41-11 | Tampering / Race | Live old-store reads during migration | mitigate | Runner defaults to copied SQLite snapshots and materialized graph/feedback export files only; live mutable store reads are rejected without a source_capture_manifest capture step. |
| T-41-12 | Tampering / Unsafe Recompute | Transform-first import path | mitigate | Runtime recompute_forbidden guard fails closed before APPLY if TEI, GLiNER, chunking, source derivation, or indexing pipeline recomputation is requested. |
