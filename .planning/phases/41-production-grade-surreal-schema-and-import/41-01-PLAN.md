<objective>
Create the production Surreal schema catalog for Phase 41.

Purpose: Phase 38 proved a thin transform-only prototype, but its schema surface is still spike-shaped. Phase 41 needs a schema-versioned catalog that can drive import, verification, and report artifacts while preserving existing production data categories where practical.

Output: A new `surreal_schema.py` module, updated `surreal.py` wiring, and focused schema tests that lock the SURR-MIG-01 storage contract before migration runner work starts.
</objective>

## Artifacts this phase produces

This plan creates or wires these Phase 41 artifacts:

- File: `backend/src/dotmd/storage/surreal_schema.py`
- Constants: `SURREAL_SCHEMA_VERSION`
- Dataclasses: `SurrealFieldDefinition`, `SurrealIndexDefinition`, `SurrealTableDefinition`, `SurrealSchemaPlan`
- Functions: `build_dotmd_surreal_schema_plan()`, `define_dotmd_surreal_schema()`, `validate_dotmd_surreal_schema_plan()`, `required_migration_categories()`
- Schema fields locked by tests: `schema_version`, `original_*` source identifiers, `chunk_strategy`, `embedding_model`, `text_hash`, `vector_rowid`, `namespace`, `document_ref`, `ref`, `active`, `bound_at`, `unbound_at`, `content_fingerprint`, `metadata_fingerprint`, `chunk_file_bindings`, `rel_type`, `weight`, `source_id`, `target_id`, `source_table`, `target_table`, `properties`, `metadata`, `checkpoint_cursor`, `source_unit_refs`
- Relation modeling: `relations` table as a metadata-carrying relation table with source/target identifiers and canonical `rel_type`; Phase 41 must not add `ENFORCED` endpoint constraints unless exporter-shape validation proves every row matches the exact endpoint set and tests cover the rejection path
- Table dispositions: `stats` is noncanonical/unsupported for migration success; `search_log`, embedding caches, and extraction caches remain unsupported categories; `vector_components` is optional/derived physical storage used only when source rows exist, not a required retrieval table
- Schema apply semantics: re-applying the same schema is idempotent; older compatible schema versions report upgrade-needed; unknown newer versions and Phase 38 `SCHEMALESS` targets fail closed with explicit recreate/replace-required status instead of silent mutation
- Existing exported wiring: `backend/src/dotmd/storage/surreal.py::define_dotmd_surreal_schema`

No package installs, runtime backend switches, retrieval implementation, shadow-run gate, production cutover, or legacy deletion are part of this plan.
