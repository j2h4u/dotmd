<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Source identifiers -> Surreal record IDs | Raw chunk IDs, refs, paths, entity names, relation IDs, and feedback IDs are untrusted identifier data. |
| Schema catalog -> Surreal target | DDL statements can mutate target schema and must be deterministic and reviewed by tests. |
| Old-store data -> schema field definitions | Source field names and category mappings must not silently omit preservation-critical data. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-41-01 | Tampering / Data Loss | `surreal_schema.py` table and field catalog | mitigate | Validate every SURR-MIG-01 category and required preservation field in RED tests before implementation. |
| T-41-02 | Tampering | `SurrealRecordIdCodec` and schema relation fields | mitigate | Keep record-ID encoding centralized in `surreal.py`; tests continue to prove raw identifiers do not shape Surreal record syntax. |
| T-41-03 | Information Disclosure | Schema metadata and preservation notes | mitigate | Schema output contains field/category metadata only, not production source refs, feedback text, or report payload rows. |
| T-41-04 | Denial of Service | Schema application | mitigate | Apply ordered DDL statements only once per explicit call; no per-request schema loading or runtime service wiring in Phase 41. |
| T-41-05 | Tampering / Unsafe Overwrite | Target schema application | mitigate | This plan defines schema only; overwrite/apply policies are delegated to Plan 41-02 and must not be implicit in schema creation. |
