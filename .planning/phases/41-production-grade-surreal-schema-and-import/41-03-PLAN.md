<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Migration reports -> operators / later phases | Reports may contain source refs, feedback messages, and graph metadata from production-derived data. |
| Devtool CLI arguments -> migration execution | Operator-provided paths and target URLs control source evidence, target writes, and report destinations. |
| Restore manifests -> safety decision | Restore evidence can incorrectly imply recoverability if counts or smoke checks are missing. |
| JSON export files -> provider/exporter adapters | Graph and feedback export files can be malformed or semantically incomplete and must fail before apply. |
| Long-running migration command -> local host resources | Production-derived imports can consume CPU, memory, and disk if not phase-bounded. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-41-11 | Information Disclosure | JSON/Markdown migration reports | mitigate | Write reports only to explicit caller paths; document that reports can include non-ASCII source refs, feedback text, and graph metadata; enforce redaction and sample-size controls. |
| T-41-12 | Tampering / Data Loss | `surreal_migration_runner.py` apply mode | mitigate | Require explicit mode, target URL, source inputs, safety gate, and overwrite policy; refuse unsafe apply invocations. |
| T-41-13 | Repudiation | Restore manifest | mitigate | Require method, CLI availability, expected/restored counts, smoke status, rehearsal target, verified status, and notes before classifying restore as verified. |
| T-41-14 | Denial of Service | Long migration commands | mitigate | Preserve named migration phases and checkpoint reporting; no service startup or per-request migration path is added. |
| T-41-15 | Tampering / Unsafe Rollback | Partial failure and rollback evidence | mitigate | `classify_surreal_migration_report()` blocks success when failed checkpoints, restore-required reports, unsafe overwrite, or missing restore evidence exist. |
| T-41-16 | Tampering / Input Validation | Graph and feedback JSON adapters | mitigate | Distinguish JSON syntax errors from semantic row validation and fail before runner invocation on invalid export files. |
| T-41-SC | Tampering | Package or `surreal` CLI installs | mitigate | No install is planned. If an executor needs to install the standalone `surreal` CLI or any new package, it must add a blocking human-verification checkpoint first because research flagged package legitimacy risk. |
</threat_model>
