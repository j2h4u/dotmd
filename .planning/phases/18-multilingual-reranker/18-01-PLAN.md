# Phase 18 Plan 01: Implement Qwen3 0.6B Multilingual Reranker

<objective>
Replace the English-oriented default reranker path with `Qwen/Qwen3-Reranker-0.6B`, selected from public benchmark, publication-age, and deployment-fit research.

The top fresh candidates are close enough in expected reranking quality that operational fit decides the first implementation target. `Qwen/Qwen3-Reranker-0.6B` wins because it is text-only, 0.6B, fresh enough, and documented through `SentenceTransformers` `CrossEncoder`. ContextualAI rerank-v2 and Jina v3 remain alternates if Qwen integration or latency fails.

This plan deliberately does not build dotMD-specific eval sets or a benchmark harness. Verification is limited to code-level tests plus an optional technical smoke against the selected serving path.
</objective>

<threat_model>
## Threat Model

| Threat | Severity | Mitigation |
|---|---:|---|
| Public benchmark results do not perfectly match dotMD's private corpus | MEDIUM | Accept this explicitly per user direction; retain `--no-rerank` and document fallback behavior. |
| Selected reranker provider is unavailable or slow | HIGH | Make reranker failure non-fatal for search: preserve fused ranking and log diagnostics. |
| Reranker score floor drops every candidate and erases valid fused results | HIGH | Change service behavior so empty reranker output falls back to fused ranking. |
| Unit tests accidentally download model weights or require a live reranker service | HIGH | Mock provider/CrossEncoder boundaries in tests. |
</threat_model>
