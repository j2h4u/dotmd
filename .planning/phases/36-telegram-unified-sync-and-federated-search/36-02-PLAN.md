<objective>
Wire the existing `ingest_application_source_runtime` into a background asyncio
polling task in `_server_lifespan`, so Telegram sync runs automatically every
`DOTMD_TELEGRAM_SYNC_INTERVAL_SECONDS` seconds (default 300) alongside the
existing `indexer_task`.

Purpose: TDD drives the polling coroutine design — the RED phase locks in the
expected interface (configurable interval, uses _local_executor, honors
shutdown_event), GREEN implements it, REFACTOR verifies lifecycle correctness.
Output: Auto-polling task that starts at server boot if Telegram is configured,
and shuts down cleanly.
</objective>

  <behavior>
    Config behavior:
    - `Settings` has `telegram_sync_interval_seconds: float = 300.0`
    - Env var `DOTMD_TELEGRAM_SYNC_INTERVAL_SECONDS` overrides the default
    - Value must be > 0

    Polling coroutine behavior:
    - Function `_run_telegram_poller(svc, bundle, interval_seconds, shutdown_event)`
      is a coroutine in mcp_server.py
    - On each iteration: calls `ingest_application_source_runtime` via
      `loop.run_in_executor(svc._local_executor, ...)` (D-LOCAL-SERIALIZED)
    - Logs one INFO line per sync: `telegram_sync discovered=N new=N changed=N
      skipped=N rebound=N failed=N`
    - On exception: logs ERROR and continues (no server crash)
    - Waits `interval_seconds` between runs using `asyncio.sleep` with early
      wakeup when `shutdown_event` is set
    - Loop terminates when `shutdown_event` is set

    Lifespan behavior:
    - When `build_if_configured("telegram")` returns a bundle: creates the polling
      task alongside `indexer_task`
    - When Telegram is not configured (returns None): no polling task created
    - At shutdown: `shutdown_event.set()` then `asyncio.wait_for(telegram_task, timeout=30)`
      (shorter than indexer's 120s — Telegram poller should exit quickly)
    - If telegram_task times out: cancel and suppress CancelledError (same pattern as indexer_task)
  </behavior>

    Step 1 — Add `telegram_sync_interval_seconds` to Settings in config.py:
      After `telegram_daemon_socket: Path | None = None` (line ~228), add:
        ```python

        ```
      This uses pydantic-settings env-var auto-mapping:
      `DOTMD_TELEGRAM_SYNC_INTERVAL_SECONDS=60` overrides the default.

    Step 2 — Add `_run_telegram_poller` coroutine in mcp_server.py:
      Add above `_server_lifespan` (or as a module-level async function):
      ```python
      async def _run_telegram_poller(
          svc: DotMDService,
          bundle: Any,
          interval_seconds: float,
          shutdown_event: asyncio.Event,
      ) -> None:
          """Background Telegram sync task — cursor-based polling at fixed interval."""
          loop = asyncio.get_running_loop()
          while not shutdown_event.is_set():
              try:
                  result = await loop.run_in_executor(
                      svc._local_executor,
                      lambda: svc._pipeline.ingest_application_source_runtime(bundle),
                  )
                  logger.info(
                      "telegram_sync discovered=%d new=%d changed=%d skipped=%d "
                      "rebound=%d failed=%d reused=%d",
                      result.discovered,
                      result.new_units,
                      result.changed_units,
                      result.skipped_units,
                      result.rebound_units,
                      result.failed_units,
                      result.reused_units,
                  )
              except Exception:
                  logger.exception("telegram_sync error during ingest")
              try:
                  await asyncio.wait_for(shutdown_event.wait(), timeout=interval_seconds)
              except TimeoutError:
                  pass
      ```

    Step 3 — Wire into `_server_lifespan` in mcp_server.py:
      After `indexer_task = asyncio.create_task(...)`, add:
      ```python
      telegram_task: asyncio.Task | None = None
      telegram_bundle = svc._source_runtime_factory.build_if_configured("telegram")
      if telegram_bundle is not None:
          telegram_task = asyncio.create_task(
              _run_telegram_poller(
                  svc,
                  telegram_bundle,
                  settings.telegram_sync_interval_seconds,
                  shutdown_event,
              )
          )
      ```

      At shutdown (after `shutdown_event.set()`), add telegram_task cleanup before
      or alongside the indexer_task wait:
      ```python
      if telegram_task is not None:
          try:
              await asyncio.wait_for(telegram_task, timeout=30)
          except TimeoutError:
              logger.warning("Telegram poller did not stop within 30s — cancelling")
              telegram_task.cancel()
              with suppress(asyncio.CancelledError):
                  await telegram_task
          except Exception:
              logger.exception("Telegram poller task failed during shutdown")
      ```
  </implementation>
