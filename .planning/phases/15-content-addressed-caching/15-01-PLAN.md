<objective>
Add a global embedding cache (text_hash + model_name → vector BLOB) in index.db so that
TEI HTTP calls are skipped for chunks whose text has already been embedded by the same model
— even after a file move that purges the chunk from vec_meta.

Purpose: File moves currently trigger full re-embedding because vec_meta rows are deleted
during purge. The embedding_cache is keyed on (text_hash, model_name) — content-addressed
and model-aware — so the vector survives the purge and is reused on the next index cycle.

Reviews addressed:
- HIGH (both reviewers): Cache key now includes model_name — stale vectors after model change
  are impossible because embedding_cache.should_invalidate() compares stored model_name against
  current settings.embedding_model and clears the table on mismatch.
- HIGH (Codex): No conn.commit() inside _embed_chunks() — deferred to the caller's existing
  commit (pipeline._save_and_embed_chunks already commits after vector store writes).
- MEDIUM (Codex): Global cache no longer silently overwrites vec_meta hits — setdefault()
  pattern ensures vec_meta wins on overlap, global cache only fills misses.

Output: storage/cache.py (new) with EmbeddingCache; pipeline.py modified to wire the cache
into _embed_chunks() and __init__().
</objective>

Create backend/src/dotmd/storage/cache.py as a new file.

The file contains one class: EmbeddingCache.

**Schema:**
```sql
CREATE TABLE IF NOT EXISTS embedding_cache (
    text_hash TEXT NOT NULL,
    model_name TEXT NOT NULL,
    embedding BLOB NOT NULL,
    created_at TEXT NOT NULL DEFAULT (datetime('now')),
    PRIMARY KEY (text_hash, model_name)
)
```

Also create a meta table for model-change invalidation:
```sql
CREATE TABLE IF NOT EXISTS embedding_cache_meta (
    key TEXT PRIMARY KEY,
    value TEXT NOT NULL
)
```

**EmbeddingCache.__init__(self, conn: sqlite3.Connection, model_name: str):**
- Store `self._conn = conn` and `self._model_name = model_name`.
- Call `self.ensure_table()` (creates tables only — does NOT write the sentinel).

**EmbeddingCache.ensure_table() -> None:**
- Execute the two CREATE TABLE IF NOT EXISTS statements above.
- Call `conn.commit()`.

**EmbeddingCache.should_invalidate(self) -> bool:**
- Query: `SELECT value FROM embedding_cache_meta WHERE key = 'model_name'`
- If no row → return False (first run, nothing to invalidate).
- If row[0] != self._model_name → return True.
- Else → return False.

**EmbeddingCache.clear(self) -> None:**
- `DELETE FROM embedding_cache`
- `DELETE FROM embedding_cache_meta`
- Upsert current model_name: `INSERT OR REPLACE INTO embedding_cache_meta VALUES ('model_name', ?)`
- `conn.commit()`

**EmbeddingCache.update_model_sentinel(self) -> None:**
- `INSERT OR REPLACE INTO embedding_cache_meta VALUES ('model_name', ?)`
- `conn.commit()`

**EmbeddingCache.lookup(self, text_hashes: list[str]) -> dict[str, list[float]]:**
- If text_hashes is empty, return {}.
- Query in batches of 500:
  `SELECT text_hash, embedding FROM embedding_cache WHERE model_name = ? AND text_hash IN (?,?,...)`
  with `self._model_name` as first bind param.
- For each row: `dim = len(blob) // 4; vec = list(struct.unpack(f"{dim}f", blob))`
- Return `{text_hash: vec}` dict.
- On any exception: log warning and return {} (graceful degradation).

**EmbeddingCache.store(self, text_hash: str, embedding: list[float]) -> None:**
- Serialize: `blob = struct.pack(f"{len(embedding)}f", *embedding)`
- `INSERT OR IGNORE INTO embedding_cache (text_hash, model_name, embedding) VALUES (?, ?, ?)`
  with `(text_hash, self._model_name, blob)`.
- Do NOT commit — caller commits in bulk.

Imports needed: `from __future__ import annotations`, `import logging`, `import sqlite3`,
`import struct`. No other imports.
