<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| live production stores -> copied snapshot | Production SQLite, feedback, and FalkorDB state is read by inventory code and copied to an offline rehearsal surface. |
| copied snapshot -> markdown reports | Store metadata becomes planning evidence that could drive a migration recommendation. |
| Docker/env inspection -> logs/reports | Environment and container details may contain sensitive values that must not be copied into reports. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-38-01-SD | Tampering | snapshot copier and inventory reports | mitigate | Record snapshot source, destination, timestamps, sizes, checksums where practical, and source mtime/row-count unchanged assertions. Treat drift as blocking evidence for import plans. |
| T-38-01-WAL | Tampering | SQLite WAL-mode snapshot copy | mitigate | Use SQLite backup API or copy/manifest `-wal` and `-shm` sidecars; incomplete sidecar handling blocks downstream migration evidence. |
| T-38-01-SQLI | Tampering | SQLite inventory dynamic table inspection | mitigate | Discover table names from `sqlite_master`, validate names before interpolation, and use parameters for values. |
| T-38-01-GRAPH | Tampering | Falkor graph inventory | mitigate | Preserve relation labels, weights, metadata keys, and edge property value types so import does not collapse graph semantics into aggregate counts. |
| T-38-01-SECRETS | Information Disclosure | Docker/env/config inventory | mitigate | Report env key names only; redact values and never serialize OAuth, Gmail, TEI, or Falkor credentials. |
| T-38-01-PARTIAL | Tampering / Availability | snapshot copy and migration map | mitigate | Mark partial snapshots incomplete; downstream plans cannot treat incomplete categories as migrated. |
| T-38-01-PARITY | Integrity | early migration feasibility scoring | mitigate | This plan cannot recommend migration; it only produces inventory evidence consumed by parity and operations gates. |
| T-38-01-SC | Tampering | dependency supply chain | mitigate | This plan does not install packages; Plan 38-02 performs package-source verification before dependency changes. |
</threat_model>
