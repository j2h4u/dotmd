## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-38-04-SD | Tampering | evidence inputs | mitigate | Recommendation cites exact prior reports and treats missing or drifted reports as failing gates. |

| T-38-04-SECRETS | Information Disclosure | operations and recommendation reports | mitigate | Redact env/config values; reference only key names and local copy paths that do not contain secrets. |
| T-38-04-PARTIAL | Tampering / Availability | restore and rollback rehearsal | mitigate | Partial import/restore status blocks migrate; restored counts must match expected imported counts before success. |
| T-38-04-CLI | Availability | backup/export/import tooling | mitigate | Use the `surreal` CLI when available; otherwise validate fallback restore correctness with reconnect, count checks, and smoke queries, or classify as defer. |
| T-38-04-ROLLBACK | Availability | rollback to current stack | mitigate | Rehearse restoring copied SQLite/sqlite-vec/FTS5 and FalkorDB originals and current-stack search/read behavior before any migrate recommendation. |
| T-38-04-SCALE | Availability | production-scale recommendation gate | mitigate | Consume Plan 38-03 record counts, HNSW build time where feasible, SurrealKV file size, and query latency; missing or failing scale evidence blocks migrate. |
| T-38-04-E2E | Integrity | cross-wave integration | mitigate | Run a same-corpus smoke through inventory, embedded gate, import, parity, operations, and recommendation assembly. |
| T-38-04-PARITY | Integrity | final recommendation | mitigate | `build_storage_recommendation` cannot return migrate unless Plan 38-03 blocking parity gate passed. |
| T-38-04-ATOM | Tampering / Availability | final recommendation | mitigate | `build_storage_recommendation` cannot return migrate unless Plan 38-05 embedded atomicity and writer-safety gate passed. |
| T-38-04-SC | Tampering | dependency supply chain | mitigate | Recommendation consumes Plan 38-05 package verification; missing verification blocks migrate. |
</threat_model>
