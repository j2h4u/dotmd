<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| copied Surreal store -> backup/restore target | Offline imported data is backed up and restored to prove rollback safety. |
| prior phase evidence -> final recommendation | Inventory, import, parity, and operations reports influence the storage decision. |
| writer process -> Surreal target path | Concurrent writers could corrupt or diverge embedded storage state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-38-04-SD | Tampering | evidence inputs | mitigate | Recommendation cites exact prior reports and treats missing or drifted reports as failing gates. |
| T-38-04-SQLI | Tampering | operations helper queries | mitigate | Operations helpers reuse Surreal storage APIs with parameterized queries; no user search text is interpolated. |
| T-38-04-SECRETS | Information Disclosure | operations and recommendation reports | mitigate | Redact env/config values; reference only key names and local copy paths that do not contain secrets. |
| T-38-04-PARTIAL | Tampering / Availability | restore and rollback rehearsal | mitigate | Partial import/restore status blocks migrate; restored counts must match expected imported counts before success. |
| T-38-04-PARITY | Integrity | final recommendation | mitigate | `build_storage_recommendation` cannot return migrate unless Plan 38-03 blocking parity gate passed. |
| T-38-04-SC | Tampering | dependency supply chain | mitigate | Recommendation consumes Plan 38-02 package verification; missing verification blocks migrate. |
</threat_model>
