<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| current stack results -> parity report | Existing SQLite/sqlite-vec/Falkor behavior becomes baseline evidence. |
| Surreal store results -> parity report | New backend behavior is compared against current behavior and may influence migration decision. |
| query strings -> Surreal search queries | User-like search terms cross into SurrealQL search expressions. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-38-03-SD | Tampering | production-derived parity sample | mitigate | Cite snapshot/import IDs from Plans 38-01 and 38-02; fail migration gate if snapshot/import evidence is missing or drifted. |
| T-38-03-SQLI | Tampering | Surreal FTS/vector query calls | mitigate | Bind search text and vector values as query parameters; do not interpolate user query text into SurrealQL strings. |
| T-38-03-SECRETS | Information Disclosure | parity report | mitigate | Report query text, chunk IDs, refs, and scores only; redact env values and connector credentials. |
| T-38-03-PARTIAL | Tampering / Availability | imported corpus used for parity | mitigate | Report imported counts beside parity cases; fail cases where imported corpus is missing categories needed by the query. |
| T-38-03-PARITY | Integrity | recommendation gate | mitigate | Blocking parity failures prevent a migrate recommendation; exact thresholds are encoded in tests and report fields. |
| T-38-03-SC | Tampering | dependency supply chain | mitigate | Depend on Plan 38-02 package verification evidence before using the Surreal SDK. |
</threat_model>
