# Plan 01 — Foundation: VecComponentStore, meta_checksum, weights config

## Objective

Create the three foundation primitives that all downstream plans depend on:

1. `VecComponentStore` — new SQLite-backed class storing raw component embedding BLOBs
2. `meta_checksum()` — new fingerprint function for title+tags-only change detection
3. `DOTMD_EMBEDDING_WEIGHTS` — env var parsed into `Settings.embedding_weights`

These are leaf-level additions: they touch no existing logic, create no regressions, and
can be implemented and tested before any pipeline refactoring begins.

**Key convention (document in module docstring):**
- `text` component: keyed by `chunk_id` (str), stores `embed(chunk.text)` as float32 BLOB
- `meta` component: keyed by canonical absolute POSIX path string (see entity_id normalization
  below), stores `embed(title + tags)` as float32 BLOB

**entity_id normalization for `meta` component (addresses Codex HIGH review concern):**

The `meta` component entity_id MUST use a canonical, normalized path string. This is critical:
if `meta` component keys use different path representations between bulk `run()` and trickle
`index_file()` (e.g., absolute vs relative, or different resolutions of symlinks), the fast
path will silently miss existing `e_meta` entries, falling back to full re-embed when it should
be a cheap metadata-only update.

Use the following canonical form:
```python
str(pathlib.Path(file_path).resolve())
```
This produces the absolute, symlink-resolved POSIX path string. The same canonical form MUST
be used by:
1. `VecComponentStore.store(entity_id, "meta", ...)` calls — caller must pass `str(Path(fp).resolve())`
2. `VecComponentStore.get(entity_id, "meta")` calls — caller must pass `str(Path(fp).resolve())`
3. `VecComponentStore.get_batch(entity_ids, "meta")` calls — caller must normalize all paths
4. `meta_tracker` FileTracker — `meta_fp_table` uses same path as key (pipeline.py ensures this)
5. `vec_meta` table path column — same canonical form

Document this invariant in the module docstring. The VecComponentStore itself does NOT
normalize entity_ids — normalization is the caller's responsibility. The docstring must
state: "For `meta` component, entity_id MUST be `str(Path(fp).resolve())`. Never pass
relative paths or unresolved symlinks."

**Data source:** `meta_checksum` calls `read_file(path)` then `parse_frontmatter(content)`,
exactly like `embed_checksum`. It reads title and tags from the parsed frontmatter dict.
It does NOT read the file path itself as a proxy — it reads the actual frontmatter.

**ADR — kind exclusion:** `meta_checksum` deliberately excludes `kind` from the hash.
Rationale: if `kind` changes, `chunk_checksum` will detect it (kind is in chunk_checksum),
triggering re-chunking which runs the full pipeline anyway (e_meta gets recomputed as
part of the full path). Therefore meta_checksum does not need to detect kind changes.
This is by design, not an oversight. Document this in the docstring.

**None safety (addresses OpenCode LOW concern):** title and tags may be None in frontmatter.
Use the same defensive pattern as `embed_checksum`: `str(frontmatter.get("title", ""))`.
For tags: guard against None with `tags or []` before iterating.

Exact implementation:
```python
def meta_checksum(path: Path) -> str:
    """Checksum for metadata-only change detection: hash(title + tags).

    ADR: meta_tracker uses this checksum to detect frontmatter-only changes
    (title renames, tag updates) without reading chunk body text.  When
    meta_checksum changes but chunk_checksum does not, the pipeline triggers
    the cheap metadata-only path: 1 TEI call (e_meta) + local fusion recompute.

    Does NOT include body text — body changes are handled by chunk_checksum.
    Does NOT include ``kind`` — intentional: kind changes are detected by
    chunk_checksum (they affect pre-split strategy), so the full re-chunk
    path runs anyway, recomputing e_meta as part of that path.

    None-safe: title=None treated as "", tags=None treated as [].
    Uses same defensive pattern as embed_checksum.

    Data source: reads frontmatter via parse_frontmatter(read_file(path)),
    same as embed_checksum.
    """
    content = read_file(path)
    frontmatter, _ = parse_frontmatter(content)
    title = str(frontmatter.get("title", "") or "")
    tags = frontmatter.get("tags", []) or []
    tags_str = ",".join(sorted(str(t) for t in tags)) if tags else ""
    return blake3.blake3(f"{title}\n{tags_str}".encode()).hexdigest()
```

Add a pydantic field_validator for `embedding_weights` that:
1. Validates format (key=value pairs)
2. Validates all values are floats
3. Validates sum ≈ 1.0 (±0.001)
4. **Requires both "text" and "meta" keys** (addresses Codex MEDIUM review concern: validator

```python
@field_validator("embedding_weights")
@classmethod
def validate_embedding_weights(cls, v: str) -> str:
    """Parse and validate embedding_weights string.

    Expected format: "text=0.7,meta=0.3"
    Raises ValueError if:
    - Any entry is not in key=value format
    - Any value is not a valid float
    - Values do not sum to 1.0 (±0.001 tolerance)
    - Either "text" or "meta" key is missing (dual-encoder requires both)
    """
    pairs = [p.strip() for p in v.split(",") if p.strip()]
    parsed: dict[str, float] = {}
    total = 0.0
    for pair in pairs:
        if "=" not in pair:
            raise ValueError(
                f"embedding_weights: invalid entry {pair!r} — expected key=value format"
            )
        key, val = pair.split("=", 1)
        key = key.strip()
        try:
            w = float(val.strip())
        except ValueError:
            raise ValueError(
                f"embedding_weights: value for {key!r} is not a float: {val!r}"
            )
        parsed[key] = w
        total += w
    if abs(total - 1.0) > 0.001:
        raise ValueError(
            f"embedding_weights: weights must sum to 1.0 (got {total:.4f})"
        )
    # Require both "text" and "meta" keys — dual-encoder architecture requires both
    # components. Accepting arbitrary keys would silently omit a component from fusion.
    if "text" not in parsed:
        raise ValueError(
            "embedding_weights: missing required key 'text' "
            "(dual-encoder requires both 'text' and 'meta')"
        )
    if "meta" not in parsed:
        raise ValueError(
            "embedding_weights: missing required key 'meta' "
            "(dual-encoder requires both 'text' and 'meta')"
        )
    return v
```

Also add a helper property to parse the string into a dict:

```python
@property
def parsed_embedding_weights(self) -> dict[str, float]:
    """Return embedding_weights as {component_name: weight} dict."""
    result: dict[str, float] = {}
    for pair in self.embedding_weights.split(","):
        pair = pair.strip()

            key, val = pair.split("=", 1)
            result[key.strip()] = float(val.strip())
    return result
```
