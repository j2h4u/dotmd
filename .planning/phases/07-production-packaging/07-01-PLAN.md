Per D-08, add a HEALTHCHECK instruction using curl. Since `python:3.12-slim` does not include curl, install it in the runtime stage.

In the runtime stage (the `FROM python:3.12-slim` block, after the existing COPY instructions and before ENTRYPOINT), add:

```dockerfile
RUN apt-get update && apt-get install -y --no-install-recommends curl && rm -rf /var/lib/apt/lists/*

HEALTHCHECK --interval=30s --timeout=5s --start-period=60s --retries=3 \
    CMD curl -f http://localhost:8000/health || exit 1
```

The `start-period=60s` accounts for model loading during warmup (sentence-transformers, GLiNER, cross-encoder take 30-60s on this CPU). During start-period, failed health checks do not count toward retries.

Place the `RUN apt-get` and `HEALTHCHECK` lines after the `ENV` declarations and `VOLUME` line, but BEFORE the `ENTRYPOINT` line. Final order in runtime stage should be:
1. COPY --from=builder (existing)
2. ENV lines (existing)
3. VOLUME (existing)
4. RUN apt-get install curl (new)
5. HEALTHCHECK (new)
6. ENTRYPOINT (existing)
