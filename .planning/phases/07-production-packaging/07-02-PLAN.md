Key design notes:
- `depends_on` references to profiled services (tei, falkordb) are silently ignored when those services are not activated (compose v5.1 behavior, verified in research). This means the same compose file works for both external and bundled modes without conditional YAML.
- `env_file: .env` per D-11 -- all configuration comes from .env, no hardcoded `environment:` block.
- `${DOTMD_PORT:-8000}` per research recommendation -- production .env sets `DOTMD_PORT=127.0.0.1:8321`.
- `${DOTMD_DATA_VOLUME:-.}` allows production to override the data volume mount source. Default `.` maps to repo-local `./data` for dev.
- The old `mcp` service is NOT included (deferred in CONTEXT.md). Can be added later with `profiles: [mcp]` if needed.
