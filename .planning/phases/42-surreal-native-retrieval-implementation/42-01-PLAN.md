<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| copied SQLite snapshot -> Surreal chunk payloads | Source metadata is parsed into retrieval fields and must not trigger source-file reads or recomputation. |
| schema/index helpers -> SurrealQL execution | Generated DDL must use fixed identifiers and validated numeric dimensions. |
| scratch capability probe -> local Surreal target | Probe statements mutate only an isolated target chosen by tests or operators. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-42-01-01 | Tampering | `build_surreal_native_retrieval_index_plan()` | mitigate | Validate `embedding_dimension` and ef inputs as positive bounded integers; keep table, field, analyzer, and index names fixed constants. |
| T-42-01-02 | Information Disclosure | `load_sqlite_rows_for_surreal()` lexical fields | mitigate | Derive title/tags only from copied snapshot rows already loaded by Phase 41; do not read arbitrary refs or source file paths. |
| T-42-01-03 | Denial of Service | capability probe | mitigate | Require scratch-target usage in tests, keep inserted probe rows tiny, and avoid production target defaults. |
| T-42-01-04 | Tampering | unsupported runtime syntax drift | mitigate | Capability report fails closed when required BM25, HNSW, or relation probes fail; newer-runtime-only features remain observations, not required gates. |
| T-42-01-SC | Tampering | package installs | accept | No npm, pip, or cargo install is planned; the existing `surrealdb` dependency is used without upgrade. |
