<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user query -> Surreal full-text predicates | Query text enters SurrealQL through variables only. |
| query embedding -> Surreal HNSW lookup | User-derived vector enters KNN query as a variable. |
| Surreal result rows -> engine tuple output | Database rows are normalized before reaching service fusion. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-42-02-01 | Tampering | `SurrealFTSSearchEngine.search()` | mitigate | Keep table/field/index identifiers fixed and pass user query as a Surreal variable; tests assert hostile query text is inert. |
| T-42-02-02 | Denial of Service | FTS and vector top-k/ef parameters | mitigate | Clamp or validate top-k and HNSW ef as bounded positive integers before statement construction. |
| T-42-02-03 | Information Disclosure | engine warning logs | mitigate | Log query length and error class, not raw query text or result snippets. |
| T-42-02-04 | Tampering | vector lookup model filter | mitigate | Filter by configured embedding model through variables so mixed-model embeddings cannot silently cross-contaminate results. |
| T-42-02-SC | Tampering | package installs | accept | No package install or SurrealDB upgrade is planned; the existing SDK/runtime is used. |
</threat_model>
