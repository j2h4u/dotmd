<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| explicit engine override -> service candidate pool | Test/dev Surreal engines can replace local engines for one collection call. |
| Surreal result sets -> Python fusion | Multiple engine outputs are combined into fused rankings and attribution. |
| fused candidates -> Phase 40 eval shape | Engine attribution must remain machine-readable for later comparison tooling. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-42-04-01 | Tampering | `engine_overrides` seam | mitigate | Accept only explicit dict keys matching known engine names; preserve defaults for absent keys; tests prove existing behavior is unchanged. |
| T-42-04-02 | Information Disclosure | hybrid attribution | mitigate | Reuse existing `SearchCandidate.matched_engines` and `engine_scores` fields; do not expose raw Surreal relation metadata or full snippets beyond current candidate hydration. |
| T-42-04-03 | Denial of Service | hybrid pool size | mitigate | Keep existing `pool_size`, `top_k`, and `fusion_k` limits from `DotMDService._collect_candidate_pool()`. |
| T-42-04-04 | Tampering | production runtime behavior | mitigate | Do not add settings, CLI flags, MCP wiring, startup code, production target defaults, or runtime fallback switches in this phase. |
| T-42-04-SC | Tampering | package installs | accept | No package install or SurrealDB upgrade is planned; existing dependencies are used. |
</threat_model>
