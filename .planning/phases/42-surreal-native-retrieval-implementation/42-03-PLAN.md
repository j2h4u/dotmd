<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user query -> entity matcher | Query text is tokenized in Python and must not shape Surreal identifiers. |
| matched entity names -> Surreal relation query | Entity names are data variables crossing into SurrealQL. |
| Surreal relation rows -> ranking | Relation weights and labels influence result ordering. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-42-03-01 | Tampering | relation query construction | mitigate | Use fixed relation-table statements and Surreal variables for entity names; tests include hostile entity text. |
| T-42-03-02 | Denial of Service | entity matching and relation lookup | mitigate | Bound n-gram matching to the existing GraphDirectEngine policy and enforce top-k limits in query/result processing. |
| T-42-03-03 | Information Disclosure | graph error logs | mitigate | Log counts and exception classes, not raw source snippets or relation metadata payloads. |
| T-42-03-04 | Tampering | relation semantics | mitigate | Filter by allowed rel_type values and use relation `weight` as numeric data; ignore malformed relation rows. |
| T-42-03-05 | Denial of Service | relation target lookup | mitigate | Use the 42-01 `relations_target_id_idx` contract by querying `target_id IN $entity_names` before bounded scoring; tests reject `scan_table()` relation scans. |
| T-42-03-SC | Tampering | package installs | accept | No package install or SurrealDB upgrade is planned; existing dependencies are used. |
</threat_model>
