<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Claude Desktop → Tailscale → /register | Dynamic registration; any Tailnet user can register a client — accepted under single-user trusted-network model |
| /authorize → redirect_uri | SDK validates redirect_uri matches registered client before provider.authorize() is called |
| Bearer token → /mcp | RequireAuthMiddleware validates token per request; provider.load_access_token() checks expiry |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-08 | Spoofing | Bearer token on /mcp | mitigate | RequireAuthMiddleware calls load_access_token() per request; provider returns None for expired/unknown tokens → 401 |
| T-17-09 | Tampering | auth_server_provider=None path | mitigate | When DOTMD_BASE_URL unset, auth=None and auth_server_provider=None; FastMCP does not add RequireAuthMiddleware — stdio and internal Docker network paths work as before without auth bypass on the HTTP port (HTTP port 8080 is not externally exposed without Tailscale) |
| T-17-10 | Elevation of Privilege | public exposure via Tailscale | mitigate | Tailscale Serve config uses "tailnet only" (confirmed: no public funnel); only Tailnet peers can reach /register and /authorize; documented in RESEARCH.md as HIGH severity, mitigated by Tailscale ACLs |
| T-17-11 | Denial of Service | token replay / flooding /authorize | accept | Single-user Tailnet; rate limiting not needed; auth codes expire in 5 minutes |
</threat_model>
