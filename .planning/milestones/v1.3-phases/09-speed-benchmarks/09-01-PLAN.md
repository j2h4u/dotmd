Create `backend/benchmarks/` directory and `bench_tei_concurrency.py`.

The script measures texts/sec for 1, 2, and 3 concurrent TEI request workers, keeping batch_size fixed at 4 (matching production DOTMD_TEI_BATCH_SIZE=4).

Structure:

1. **`generate_test_texts(n=100, avg_words=150) -> list[str]`**
   - Generate n synthetic texts of ~avg_words length using a fixed seed paragraph about server infrastructure, knowledge bases, and search (realistic tokenization). Vary length slightly per text (+/- 10 words based on index modulo) for natural distribution.
   - Use a fixed base paragraph:
     ```
     "The server infrastructure runs on a dedicated machine with Docker containers. "
     "Each service communicates through internal networks using REST APIs. "
     "The knowledge base contains transcribed voice notes and documentation files. "
     "Search combines semantic embeddings, keyword matching, and graph traversal. "
     ```
   - Repeat/slice words from this base to reach target length per text.

2. **`embed_batch(client: httpx.Client, url: str, texts: list[str]) -> int`**
   - POST to `{url}/embed` with `json={"inputs": texts, "truncate": True}`, timeout=120.0
   - `raise_for_status()`, return `len(texts)`

3. **`benchmark_concurrency(url: str, texts: list[str], batch_size: int, max_workers: int, warmup_batches: int = 2) -> float`**
   - Split texts into batches of `batch_size` using list slicing `texts[i:i+batch_size]`
   - Warmup: run `warmup_batches` batches sequentially (same client)
   - Timed run: use `concurrent.futures.ThreadPoolExecutor(max_workers=max_workers)` to submit all batches in parallel. Each future calls `embed_batch`. Sum results from `as_completed(futures)`. Measure wall-clock with `time.perf_counter()`.
   - Return `total_texts / elapsed_seconds`

4. **`main()`**
   - Read `DOTMD_EMBEDDING_URL` from env, default `"http://embeddings:80"`
   - Read `DOTMD_TEI_BATCH_SIZE` from env as int, default `4`
   - Generate 100 test texts
   - For each worker count in `[1, 2, 3]`: run 3 iterations of `benchmark_concurrency`, collect texts/sec for each iteration
   - Compute `mean`, `stdev`, `min`, `max` using `statistics.mean`, `statistics.stdev`, `statistics.fmean` (use `statistics` stdlib)
   - Print header block:
     ```
     TEI Concurrency Benchmark
     =========================
     Embedding URL: {url}
     Test corpus: 100 texts, ~150 words each
     Batch size per request: {bs}
     Warmup: 2 batches
     Iterations: 3
     ```
   - Print results table:
     ```
     | Workers | texts/sec (mean) | stddev | min    | max    |
     |---------|------------------|--------|--------|--------|
     | 1       | 12.3             | 0.4    | 11.8   | 12.7   |
     ```
   - Determine best worker count (highest mean texts/sec)
   - Compute speedup vs workers=1 (best_mean / sequential_mean)
   - Print conclusion:
     ```
     CONCLUSION: Concurrency {helps|does not help}. Best throughput at workers={N}.
     Speedup vs sequential: {X.Xx}
     ```
   - "helps" if best worker count > 1 AND speedup > 1.05 (5% threshold to avoid noise)

5. **`if __name__ == "__main__": main()`** guard

Imports: `os`, `time`, `statistics`, `concurrent.futures` (ThreadPoolExecutor, as_completed), `httpx`

Do NOT import anything from dotmd package -- this is a standalone script.
Do NOT touch production indexes or `~/.dotmd/`.
Do NOT use asyncio -- ThreadPoolExecutor is sufficient for 1-3 workers of I/O-bound HTTP calls.

Create `backend/benchmarks/bench_gliner_batching.py`.

The script measures texts/sec for sequential NER (predict_entities per text) vs batch NER (inference with batch_size=1,4,8) and optionally with sequence packing.

Structure:

1. **`generate_test_texts(n=50, avg_words=150) -> list[str]`**
   - Same approach as TEI benchmark: fixed seed paragraph repeated/sliced to target length, slight per-text variation.
   - Use 50 texts (not 100) because GLiNER is slower (~1 text/sec on this CPU).

2. **`ENTITY_TYPES`** module-level constant:
   ```python
   ENTITY_TYPES = ["person", "organization", "technology", "concept", "location"]
   ```

3. **`MODEL_NAME`** module-level constant: `"urchade/gliner_multi-v2.1"`

4. **`THRESHOLD`** module-level constant: `0.5`

5. **`benchmark_sequential(model, texts: list[str]) -> float`**
   - Loop over texts, call `model.predict_entities(text, ENTITY_TYPES, threshold=THRESHOLD)` for each
   - Measure wall-clock with `time.perf_counter()`
   - Return `len(texts) / elapsed`

6. **`benchmark_batch(model, texts: list[str], batch_size: int) -> float`**
   - Call `model.inference(texts, ENTITY_TYPES, threshold=THRESHOLD, batch_size=batch_size)`
   - Measure wall-clock
   - Return `len(texts) / elapsed`

7. **`benchmark_batch_packed(model, texts: list[str], batch_size: int) -> float`**
   - Try to import `from gliner.infer_packing import InferencePackingConfig`
   - If import fails, return -1.0 (packing not available in this version)
   - Create `packing = InferencePackingConfig(max_length=512, streams_per_batch=1)`
   - Call `model.inference(texts, ENTITY_TYPES, threshold=THRESHOLD, batch_size=batch_size, packing_config=packing)`
   - Measure wall-clock
   - Return `len(texts) / elapsed`

8. **`main()`**
   - Load model: `from gliner import GLiNER; model = GLiNER.from_pretrained(MODEL_NAME)`
   - Print "Model loaded." after load completes
   - Generate 50 test texts
   - Warmup: run `model.predict_entities` on first 5 texts (discard results, printed as "Warmup complete.")
   - Define benchmark configs as list of tuples: `(label, function, kwargs)`
     ```python
     configs = [
         ("Sequential",    benchmark_sequential, {"texts": texts}),
         ("Batch bs=1",    benchmark_batch, {"texts": texts, "batch_size": 1}),
         ("Batch bs=4",    benchmark_batch, {"texts": texts, "batch_size": 4}),
         ("Batch bs=8",    benchmark_batch, {"texts": texts, "batch_size": 8}),
         ("Packed bs=8",   benchmark_batch_packed, {"texts": texts, "batch_size": 8}),
     ]
     ```
   - For each config: run 3 iterations, collect texts/sec. If function returns -1.0 (packing unavailable), print "N/A" for that row and skip.
   - Compute mean, stdev, min, max using `statistics` stdlib
   - Print header block:
     ```
     GLiNER Batching Benchmark
     =========================
     Model: urchade/gliner_multi-v2.1
     Entity types: person, organization, technology, concept, location
     Test corpus: 50 texts, ~150 words each
     Warmup: 5 texts
     Iterations: 3
     ```
   - Print results table:
     ```
     | Mode            | batch_size | texts/sec (mean) | stddev | min  | max  |
     |-----------------|------------|------------------|--------|------|------|
     | Sequential      | 1          | 0.8              | 0.05   | 0.7  | 0.85 |
     ```
   - Determine best mode (highest mean texts/sec, excluding N/A rows)
   - Compute speedup vs sequential (best_mean / sequential_mean)
   - Print conclusion:
     ```
     CONCLUSION: Batching {helps|does not help}. Best throughput at {best_label}.
     Speedup vs sequential: {X.Xx}
     ```
   - "helps" if best mode is not "Sequential" AND speedup > 1.05

9. **`if __name__ == "__main__": main()`** guard

Imports: `time`, `statistics`, and `gliner` (GLiNER, optionally InferencePackingConfig).

Do NOT import anything from dotmd package.
Do NOT use deprecated `batch_predict_entities` method.
Do NOT touch production indexes or `~/.dotmd/`.

Note on memory safety: batch_size capped at 8 to avoid OOM on the 16GB server (research pitfall 3). Texts are ~150 words (~200 tokens), well within GLiNER's 512-token window.
