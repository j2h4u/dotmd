    **New class: `FTS5SearchEngine`** (replaces `BM25SearchEngine`):

    1. **Constructor** `__init__(self, conn: sqlite3.Connection)`:
       - Takes a sqlite3.Connection (shared from metadata store, already in WAL mode)
       - Creates FTS5 virtual table if not exists:
         ```sql
         CREATE VIRTUAL TABLE IF NOT EXISTS chunks_fts USING fts5(
             chunk_id UNINDEXED,
             text,
             tokenize = 'unicode61'
         )
         ```
       - Commits after creation.

    2. **`add_chunks(self, chunks: list[Chunk]) -> None`** (new method for incremental insert):
       - Extract `(chunk_id, text)` tuples from chunks list
       - Use `INSERT OR REPLACE INTO chunks_fts(chunk_id, text) VALUES (?, ?)`
       - executemany + commit
       - Log at DEBUG level: "FTS5: added %d chunks"

    3. **`remove_chunks(self, chunk_ids: list[str]) -> None`** (new method for incremental delete):
       - For each chunk_id: `DELETE FROM chunks_fts WHERE chunk_id = ?`
       - Commit once after all deletes
       - Log at DEBUG level: "FTS5: removed %d chunks"

    4. **`search(self, query: str, top_k: int = 10) -> list[tuple[str, float]]`**:
       - Sanitize query using `_sanitize_fts5_query()` helper
       - Execute: `SELECT chunk_id, -rank AS score FROM chunks_fts WHERE chunks_fts MATCH ? ORDER BY rank LIMIT ?`
       - Catch `sqlite3.OperationalError`, log warning, return empty list
       - Return `[(chunk_id, score), ...]`

    5. **`load_index(self) -> None`** (compatibility method):
       - Check if `chunks_fts` is empty but `chunks` table has data
       - If so, run one-time migration: `INSERT INTO chunks_fts(chunk_id, text) SELECT chunk_id, text FROM chunks`
       - Log: "FTS5: migrated %d chunks from metadata"
       - If both empty or FTS5 already populated, log info and return

    6. **`build_index(self, chunks: list[Chunk]) -> None`** (compatibility wrapper):
       - Call `self.add_chunks(chunks)` -- keeps backward compatibility with pipeline's current call pattern
       - This is a temporary bridge; pipeline.py will be updated in this same plan

    7. **Helper `_sanitize_fts5_query(query: str) -> str`**:
       - Module-level function
       - Use regex to remove FTS5 special chars: `["\(\)\*:]`
       - Split into words, wrap each in double quotes: `'"word1" "word2"'`
       - Return empty string if no words remain

    **Keep**: `from __future__ import annotations` at top, module-level `logger = logging.getLogger(__name__)`
