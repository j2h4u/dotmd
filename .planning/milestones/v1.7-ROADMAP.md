# Milestone v1.7: Storage Simplification

**Status:** Shipped 2026-06-12
**Phases:** 38
**Total Plans:** 5

## Overview

v1.7 evaluated whether Embedded SurrealDB could replace dotMD's split
SQLite/sqlite-vec/FTS5 + FalkorDB storage stack while migrating existing data
instead of recomputing chunks, embeddings, and extracted entities.

Phase 38 produced a conservative `reject` recommendation for the first
compatibility/parity-style prototype, but it also proved that transform-first
data movement and embedded operations are plausible enough to justify a new
SurrealDB-native cutover milestone.

## Phases

### Phase 38: Embedded SurrealDB storage spike

**Goal:** Decide whether dotMD should replace separate SQLite/sqlite-vec/FTS5
and FalkorDB storage with one embedded SurrealDB-backed storage layer while
migrating existing production data wherever safe.

**Plans:** 5

- [x] 38-01: Current storage inventory and migration map
- [x] 38-02: Transform-only Surreal import proof
- [x] 38-03: Retrieval parity harness and evidence
- [x] 38-04: Operations evidence and final recommendation
- [x] 38-05: Embedded SurrealDB safety gate

## Milestone Summary

**Key accomplishments:**

- Captured WAL-safe copied SQLite snapshots plus Falkor/feedback inventory.
- Proved stored chunks, vectors, graph rows, and feedback can be represented in
  a thin SurrealDB prototype without default recomputation.
- Verified embedded `surrealkv://` transaction behavior and local writer guard
  behavior on copied/local stores.
- Built a retrieval comparison harness for full-text, vector, graph-direct, and
  hybrid/RRF behavior.
- Produced a final reject recommendation for the first migrate-ready attempt
  because weighted FTS and hybrid/RRF evidence did not support cutover.

**Decision at close:**

The next effort should not chase exact old-stack compatibility. v1.8 should
define a SurrealDB-native retrieval contract, prove quality with golden queries
and shadow runs, cut production over, and then delete the legacy stack.

**Issues deferred:**

- Weighted SurrealDB full-text design with title/tags/body contributions.
- SurrealDB-native graph traversal on production-derived relation data.
- Production-scale vector index build/latency evidence.
- Legacy SQLite/sqlite-vec/FTS5/FalkorDB/LadybugDB removal after cutover.
