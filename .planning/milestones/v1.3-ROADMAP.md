# Milestone v1.3: Production Packaging & Background Indexing

**Status:** SHIPPED 2026-03-28
**Phases:** 7-10
**Total Plans:** 8

## Overview

Production-ready packaging with parameterized Docker Compose, external smoke tests, performance benchmarks, and a background trickle indexer that processes new files incrementally without blocking search queries.

## Phases

### Phase 7: Production Packaging

**Goal**: Docker compose stack with health checks, WAL mode, parameterized config, and production overlay
**Depends on**: Phase 6 (Docker Integration + Migration)
**Plans**: 2 plans

Plans:

- [x] 07-01: Health Endpoint & Docker Readiness — /health liveness probe, HEALTHCHECK with curl, WAL mode for vec.db
- [x] 07-02: Docker Compose Stack — Parameterized compose with TEI/FalkorDB bundled profile, .env.example, production include-based deploy

**Details:**
- GET /health liveness endpoint (no dependency checks)
- WAL journal mode on all SQLite databases for concurrent access
- Dockerfile HEALTHCHECK with 60s start-period for model warmup
- Parameterized docker-compose.yml with env vars and bundled profile
- .env.example documenting all 25+ DOTMD_* variables
- Production /opt/docker/dotmd/ with include directive referencing repo compose
- Compose profile pattern: `--profile bundled` activates TEI + FalkorDB

### Phase 8: Smoke Tests

**Goal**: External HTTP smoke tests validating all search engines and API structure
**Depends on**: Phase 7 (Production Packaging)
**Plans**: 1 plan

Plans:

- [x] 08-01: Smoke Test Suite — 5 tests covering semantic/BM25/graph engines, hybrid fusion, and API response schema

**Details:**
- Session-scoped httpx client with skip-on-unavailable hook
- Individual engine tests validating matched_engines field
- Hybrid fusion test confirming multi-engine results (top_k=50)
- API structure test validating response schema
- pytest.ini isolation from parent conftest (no dotmd imports needed)

### Phase 9: Speed Benchmarks

**Goal**: Measure TEI concurrency and GLiNER batching performance to inform indexer parameters
**Depends on**: Phase 4 (TEI integration pattern)
**Plans**: 1 plan

Plans:

- [x] 09-01: TEI Concurrency & GLiNER Batching Benchmarks — Standalone scripts measuring throughput for concurrent/batch configurations

**Details:**
- TEI concurrency benchmark: 1/2/3 workers, fixed batch_size, ThreadPoolExecutor
- GLiNER batching benchmark: sequential vs batch (bs=1,4,8) vs sequence packing
- Results: concurrent TEI gives no gain (TEI saturates all cores on single request), GLiNER batching slower than sequential
- Both scripts: synthetic data, warmup, 3 iterations, stats table with CONCLUSION

### Phase 10: Background Trickle Indexer

**Goal**: Background indexer processes new files one at a time while API serves search queries
**Depends on**: Phase 5 (BM25), Phase 7 (compose stack)
**Plans**: 4 plans

Plans:

- [x] 10-01: FTS5 BM25 Replacement — SQLite FTS5 replaces rank_bm25+pickle for incremental keyword search
- [x] 10-02: Config & File Discovery — TOML config source, multi-path discovery with glob/exclude
- [x] 10-03: Trickle Indexer Core — Background loop with watchdog filesystem watching, per-file pipeline in thread pool
- [x] 10-04: Status Endpoint & CLI Progress — Trickle progress fields in IndexStats, API /status, CLI display

**Details:**
- FTS5SearchEngine with incremental add_chunks/remove_chunks (removed rank-bm25 dependency)
- Settings loads from ~/.dotmd/config.toml with env var override priority
- TrickleIndexer: backlog processing (newest mtime first) then watch mode
- Watchdog inotify + hourly polling fallback
- Per-file processing through full pipeline in asyncio.to_thread
- Graceful shutdown with 120s timeout
- Status reporting: backlog count, rate (files/hr), ETA

## Milestone Summary

**Key Decisions:**

- Health endpoint is liveness-only (no dependency checks) — fast and independent
- Compose profiles for optional bundled services (TEI, FalkorDB)
- Production include directive references repo compose as single source of truth
- FTS5 shares metadata store SQLite connection (WAL mode) instead of separate file
- unicode61 tokenizer for bilingual RU/EN FTS5 support
- Per-file trickle processing in thread pool to avoid blocking event loop
- 2-second debounce on watchdog events for rapid file saves

**Issues Resolved:**

- Compose v5.1 depends_on errors on profiled services (removed depends_on)
- Compose port list merge is append-only (moved to env var interpolation)
- Parent conftest isolation for smoke tests (pytest.ini boundary)
- Hybrid fusion top_k too low for multi-engine results (increased to 50)

**Issues Deferred:**

- None — all v1.3 issues resolved during milestone

**Technical Debt Incurred:**

- bm25_path property in config.py kept as dead code (harmless)
- build_index() backward-compatible wrapper in FTS5SearchEngine
