  <action>
Create `backend/scripts/eval_compare.py` -- a standalone script that:

1. Loads a saved baseline JSON file (from eval_baseline.py output) via `--baseline` arg.

2. Runs the same queries against the current service (same HTTP approach as eval_baseline.py).

3. For each query, computes and reports:
   - **Count delta**: baseline count vs current count
   - **Rank changes**: for each chunk_id present in both, show old_rank -> new_rank
   - **New hits**: chunk_ids in current but not baseline
   - **Lost hits**: chunk_ids in baseline but not current
   - **Score delta**: for shared hits, show score change (current - baseline)
   - **Top-3 stability**: are the top-3 chunk_ids the same?

4. Prints a per-query comparison table to stdout:
```
=== q1_semantic_concept: "распределение прибыли" ===
Count: 5 -> 4 (-1)
Top-3 stable: NO (2/3 match)

  Rank  Baseline                          Current                           Delta
  1     [0.885] Рабочая встреча...        [0.912] Рабочая встреча...        +0.027
  2     [0.861] Рабочая встреча...        [0.845] Рабочая встреча...        -0.016
  ...

New hits: chunk_abc (score 0.801)
Lost hits: chunk_xyz (was rank 5, score 0.630)
```

5. At the end, prints a summary verdict:
```
SUMMARY:
  Queries improved: 2/5
  Queries degraded: 1/5
  Queries unchanged: 2/5
  Top-3 stability: 3/5 queries
  Average score delta: +0.034
```

A query is "improved" if: (a) count of relevant hits increased, or (b) top-3 scores are higher on average, or (c) previously noisy results (rank 3-5) dropped out. A query is "degraded" if top-3 hits are lost or scores dropped significantly (>0.05). Otherwise "unchanged".

6. Optionally saves full comparison to `--output` JSON file.

Use `argparse`: `--baseline` (required), `--host` (default `http://localhost:8000`), `--output` (optional), `--top-k` (default 10).

Add shebang `#!/usr/bin/env python3`. No dotmd imports.
