## Fix 1 — pipeline.py: collapse [prof] lines into one DONE summary per file

**File:** `backend/src/dotmd/ingestion/pipeline.py`

**What to change in `_index_file`:**

Current structure (under `if prof:` guards, scattered): 7 separate logger.info calls
emitting `[prof] {path} purge: …`, `chunk:`, `save+fts5:`, `extraction:`, `graph:`,
`embed:`, `vec_store:`, plus a final `TOTAL:` line.

New structure:

1. At the top of `_index_file` (before phase 1), always capture `t_file = time.perf_counter()`
   (unconditional, not gated on `prof`).

2. For each timed phase, collect the elapsed into a local var instead of logging immediately.
   The `if prof: t0 = time.perf_counter()` / `if prof: logger.info(...)` pairs become:
   - `t0 = time.perf_counter()` (unconditional)
   - `t_purge = time.perf_counter() - t0` (unconditional, at end of purge block)
   - same pattern for `t_chunk`, `t_save`, `t_extract`, `t_graph`, `t_embed`, `t_vec`

3. The per-phase `if prof: logger.info("[prof] ...")` lines are REMOVED entirely.

4. The existing `if prof: logger.info("[prof] %s TOTAL: ...")` at the end is REPLACED
   with ONE unconditional INFO line emitted at the very end of `_index_file`
   (after all phases, just before `return`):

   ```python
   t_total = time.perf_counter() - t_file
   logger.info(
       "pipeline: %s DONE %d chunks %.1fs"
       " (chunk %.2f / save %.2f / extract %.2f / graph %.2f / embed %.2f / vec %.2f)",
       file_info.path.name,
       len(chunks) if chunks else 0,
       t_total,
       t_chunk, t_save, t_extract, t_graph, t_embed, t_vec,
   )
   ```

   For phases that were skipped (e.g., no `needs_embed` so embed never ran), their
   timing vars are 0.0. Initialise all 7 timing vars to 0.0 at the top of `_index_file`
   so they are always defined at the return point regardless of code path:
   `t_purge = t_chunk = t_save = t_extract = t_graph = t_embed = t_vec = 0.0`

5. `_beacon()` calls are unchanged — they remain gated on `prof` (file-based, not log-based).

**Resulting output (profile-off):** 1 INFO line per file.
**Resulting output (profile-on):** still 1 INFO line per file (the per-phase detailed lines are gone; the breakdown is now inline in the DONE line).

**Commit message:** `fix(quick): pipeline — collapse [prof] per-phase lines into one DONE summary`

## Fix 3 — search/semantic.py: throttle TEI heartbeat to 30s intervals

**File:** `backend/src/dotmd/search/semantic.py`

**Current code** in `_encode_via_tei` (around line 147-172):
```python
log_interval = max(1, min(50, total_batches // 20))  # ~20 log lines total

for batch_idx, i in enumerate(range(0, len(inputs), bs)):
    ...
    if batch_idx % log_interval == 0 or batch_idx == total_batches - 1:
        done = batch_idx + 1
        elapsed = time.perf_counter() - t_start
        rate = done / elapsed if elapsed > 0 else 0
        remaining = (total_batches - done) / rate if rate > 0 else 0
        ...
        logger.info("TEI %d/%d ...")
```

**New behaviour:**

1. Before the loop, emit one unconditional start line:
   ```python
   logger.info("TEI start: %d chunks (%d batches, bs=%d)", len(inputs), total_batches, bs)
   ```

2. Replace the `log_interval` counter-based gate with a wall-clock gate:
   ```python
   t_last_heartbeat = t_start  # initialised before loop

   # inside loop, after results.extend(...):
   now = time.perf_counter()
   is_last = (batch_idx == total_batches - 1)
   if is_last or (now - t_last_heartbeat) >= 30.0:
       done = batch_idx + 1
       elapsed = now - t_start
       rate = done / elapsed if elapsed > 0 else 0
       remaining = (total_batches - done) / rate if rate > 0 else 0
       if remaining < 60:
           eta = f"ETA ~{remaining:.0f}s"
       else:
           eta = f"ETA ~{remaining / 60:.1f}min"
       logger.info(
           "TEI %d/%d (%.0f%%) %.1f batches/s, %s",
           done, total_batches, done / total_batches * 100, rate, eta,
       )
       t_last_heartbeat = now
   ```

3. Remove the `log_interval` variable entirely (it's replaced by wall-clock).

4. The existing `logger.info("TEI complete: ...")` line after the loop stays unchanged
   (it's the unconditional end line).

**Result:** for a 60-chunk file at bs=32 (2 batches, ~5s total): start + end = 2 lines.
For a 500-chunk file taking 3 min: start + ~6 heartbeats + end.

**Commit message:** `fix(quick): semantic — throttle TEI heartbeat to 30s wall-clock intervals`

## Fix 4 — trickle.py: suppress early ETA noise

**File:** `backend/src/dotmd/ingestion/trickle.py`

**Location:** `_process_backlog` method, the progress log block (~lines 400-417) and
`_update_eta` method (~lines 561-570).

**Current `_update_eta`:** always computes `eta_minutes` once `files_done > 0`.

**Change `_update_eta`:** suppress ETA until both conditions are met:
```python
def _update_eta(self, files_done: int, files_total: int) -> None:
    """Update throughput rates and ETA.

    ETA is suppressed until at least 3 files have been processed AND
    at least 120 seconds have elapsed, to avoid wildly inflated early estimates.
    """
    elapsed = time.monotonic() - self._state._start_time
    if elapsed > 0 and files_done > 0:
        self._state.files_per_hour = files_done / (elapsed / 3600)
        if self._state.total_chunks_done > 0:
            self._state.chunks_per_hour = (
                self._state.total_chunks_done / (elapsed / 3600)
            )
        # Suppress ETA until sample is large enough to be meaningful
        if files_done >= 3 and elapsed >= 120:
            remaining = files_total - files_done
            if self._state.files_per_hour > 0:
                self._state.eta_minutes = (
                    remaining / (self._state.files_per_hour / 60)
                )
        else:
            self._state.eta_minutes = None  # "estimating..."
```

**Change the ETA format string in `_process_backlog`** progress log block:
```python
eta_str = ""
if self._state.eta_minutes is not None:
    if self._state.eta_minutes < 60:
        eta_str = f", ETA ~{self._state.eta_minutes:.0f}min"
    else:
        eta_str = f", ETA ~{self._state.eta_minutes / 60:.1f}hr"
else:
    # Still collecting sample — don't lie
    eta_str = ", ETA: estimating..."
```

(The `else` branch is new — previously nothing was emitted when eta_minutes was None.)

**Commit message:** `fix(quick): trickle — suppress ETA until 3 files + 120s threshold`
