# Quick 260425-m79: Log Clarity Kaizen 6-Point Cleanup Summary

## What Changed Per Fix

**Fix 1 — pipeline.py `_index_file`:**
- All 7 `if prof: logger.info("[prof] ...")` calls removed
- Timing vars (`t_purge`, `t_chunk`, `t_save`, `t_extract`, `t_graph`, `t_embed`, `t_vec`) now always collected (unconditional), initialised to 0.0
- Single unconditional INFO line emitted at end: `pipeline: {basename} DONE N chunks X.Xs (chunk / save / extract / graph / embed / vec)`
- `_beacon()` calls unchanged (still gated on `prof`)

**Fix 2 — trickle.py `_process_one_file`:**
- No code removed (only one INFO + one DEBUG already present)
- Docstring updated to document the two-line-per-file contract explicitly

**Fix 3 — semantic.py `_encode_via_tei`:**
- `log_interval` counter removed
- Unconditional start line added: `TEI start: N chunks (B batches, bs=S)`
- Loop gate: `is_last OR (now - t_last_heartbeat) >= 30.0`
- For small batches (< 30s total): start + end = 2 lines

**Fix 4 — trickle.py `_update_eta` + progress block:**
- ETA suppressed when `files_done < 3` or `elapsed < 120s`; `eta_minutes` set to None in that case
- Progress line now shows `, ETA: estimating...` instead of nothing when suppressed

**Fix 5 — trickle.py `_process_backlog`:**
- Backlog summary line now appends ` — queued: file1.md, file2.md, ...` (max 5 shown + overflow count)
- Per-file `queued: {path} (mtime ...)` block removed entirely

**Fix 6 — trickle.py `TrickleState` + `_process_backlog`:**
- `last_indexed_at: datetime | None = None` added to TrickleState
- Set after `succeeded += 1` in `_process_backlog` and after `indexed_count += 1` in `_watch_mode`
- Zero-changes backlog scan emits: `Backlog: idle (N indexed, last activity YYYY-MM-DDTHH:MM:SSZ)`
